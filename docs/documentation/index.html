Checkpoint Documentation

Checkpoint is a save game manager for local games. It helps you create backups (snapshots) of your save files, switch between different saves, and protect your progress from accidents or corruption.

What is Checkpoint?

If you play games outside of Steam, GOG, or other platforms that automatically sync saves, you probably know the pain of losing progress. Maybe a save got corrupted, or you want to try a different path in the game without losing your main progress.

Checkpoint solves this by letting you create timestamped backups of your save files. You can have multiple saves for the same game, switch between them instantly, and never worry about losing hours of progress again.

Key Features

  • Snapshots - Create timestamped backups with one click
  • Instant Restore - Switch between saves in seconds
  • Smart Protection - Auto-backs up current save before restoring
  • Game Covers - Upload custom images to identify games visually
  • Process Detection - Won't restore while game is running
  • Cross-Platform - Windows, Linux, and macOS support

Installation

Windows

  1. Download checkpoint.exe from the releases page
  2. Place it anywhere (Documents, Desktop, etc.)
  3. Double-click to run - no installation needed

Checkpoint is a portable executable. It doesn't require installation and doesn't write to the registry.

Linux

Debian/Ubuntu:

sudo dpkg -i checkpoint_0.1.0_amd64.deb

Fedora:

sudo rpm -i checkpoint-0.1.0-1.x86_64.rpm

Arch (AUR):

yay -S checkpoint

You can also run the AppImage directly without installing: chmod +x Checkpoint-0.1.0.AppImage && ./Checkpoint-0.1.0.AppImage

macOS

  1. Download the Checkpoint_0.1.0.dmg
  2. Open the DMG and drag Checkpoint to your Applications folder
  3. On first launch, you may need to right-click and select "Open" due to Gatekeeper

Your First Game

Let's add your first game to Checkpoint. For this example, we'll use a hypothetical game called "RPG Adventure" that stores saves in Documents/MyGames/RPGAventure.

Step 1: Click "Add Game"

In the sidebar, click the green "Add Game" button. This opens a modal where you'll enter the game details.

Step 2: Fill in the Details

  • Game Name - "RPG Adventure" (whatever you want to call it)
  • Save Location - The folder where save files are stored. Use the "Browse" button to navigate to it, or type the path directly.
  • Executable Name (optional) - "RPGAdventure.exe" - This lets Checkpoint check if the game is running before restoring saves.

Step 3: Add a Cover (Optional)

Click on the cover upload area to add an image. This helps identify the game visually. Works best with vertical images (3:4 ratio) like game box art. Max size 2MB, JPG or PNG.

Step 4: Save

Click "Add Game" and you're done! The game now appears on your dashboard with a card showing the cover (or a placeholder), name, save location, and creation date.

Working with Snapshots

Snapshots are the heart of Checkpoint. They're complete copies of your save folder at a specific point in time, stored safely in your backup location.

Creating a Snapshot

  1. Click on a game card to open its detail page
  2. Click the "Create Snapshot" button
  3. Give it a name (optional) - something descriptive like "Before Boss Fight" or "Good Ending Route"
  4. Click "Create"

The snapshot is created with a timestamp. For large games (several GB), this might take a minute. A loading indicator shows progress.

Snapshot Details

Each snapshot shows:

  • Name (or timestamp if unnamed)
  • Date and time created
  • Size of the backup
  • Number of files

Renaming Snapshots

Click the pencil icon on any snapshot to rename it. This is useful when you initially create a quick backup but later want to give it a more descriptive name.

Deleting Snapshots

Click the trash icon to delete a snapshot. You'll get a confirmation dialog because this cannot be undone. Deleted snapshots free up disk space immediately.

Restoring Saves

Restoring replaces your current save files with the files from a snapshot. This is useful for:

  • Going back to an earlier point in the game
  • Trying different choices/paths without starting over
  • Recovering from corrupted saves
  • Sharing saves between different playthroughs

How to Restore

  1. Open the game's detail page
  2. Find the snapshot you want to restore
  3. Click the restore button (circular arrow icon)
  4. Confirm the restore operation

Safety Features

Auto-Backup: Before restoring, Checkpoint automatically creates a snapshot of your current save. This means if you change your mind, you can restore back to where you were. Look for snapshots named "Auto-backup before restore" in your snapshot list.

Process Detection: If you provided an executable name when adding the game, Checkpoint will check if it's running before allowing a restore. This prevents conflicts and corrupted saves. Close the game first, then restore.

Warning: Restoring will overwrite your current save files. While Checkpoint auto-backs up first, make sure you understand which snapshot you're restoring to. The confirmation dialog shows the snapshot name and date to help prevent mistakes.

Game Covers

Game covers make your dashboard look great and help you quickly identify games. They're optional but recommended.

Best Practices

  • Aspect Ratio: Use vertical images (portrait orientation), ideally 3:4 ratio
  • Size: Keep under 2MB for fast loading
  • Format: JPG for photos, PNG for graphics with transparency
  • Source: Box art from SteamGridDB, IGDB, or official game sites work great

Adding Covers to Existing Games

Currently, covers can only be added when creating a game. If you want to add a cover to an existing game, you'll need to:

  1. Note down the save location from the game's detail page
  2. Delete the game from Checkpoint (this doesn't delete your actual saves)
  3. Add the game again with the same details plus the cover image

Future versions will support adding covers to existing games directly.

Managing Games

Deleting a Game

To remove a game from Checkpoint:

  1. Open the game's detail page
  2. Click the "Delete Game" button (top right)
  3. Confirm the deletion

Important: Deleting a game from Checkpoint removes the game entry and all its snapshots from your backup location. It does NOT delete your actual game saves from their original location. Your current progress remains untouched.

Where Are Backups Stored?

By default, snapshots are stored in:

  • Windows: C:\Users\YourUsername\checkpoint\
  • Linux/macOS: ~/checkpoint/ (your home folder)

Inside that folder, each game gets its own folder named with the game's ID, and snapshots are stored within those folders.

You can change this location in Settings if you want backups on a different drive (useful if you have limited space on your main drive).

Settings

Access settings by clicking "Settings" in the sidebar. Here you can customize Checkpoint to your preferences.

Appearance

Toggle between Light and Dark themes. Your preference is saved and applies immediately across the app.

Language

Checkpoint supports multiple languages:

  • English
  • Portuguese
  • Spanish

Select your preferred language from the dropdown. The change applies immediately.

Backup Location

By default, Checkpoint stores all snapshots in ~/checkpoint/ on your home drive. You can change this to any folder you have write access to.

Why Change It?

  • Different Drive: Store backups on an external drive or secondary HDD/SSD
  • Cloud Sync: Point to a Dropbox, Google Drive, or OneDrive folder for automatic cloud backup
  • Space Management: If your main drive is filling up, move backups elsewhere

Changing the Location

  1. Open Settings
  2. Under "Backup Location", click "Browse" or type a new path
  3. Click "Save Settings"
Warning: Changing the backup location does NOT move existing snapshots. You'll need to manually move the files from the old location to the new one if you want to keep them. New snapshots will go to the new location.

Recommended Setup

For the best protection, consider pointing your backup location to:

  • An external drive that you only connect for backups
  • A cloud-synced folder (Dropbox, Google Drive, etc.)
  • A NAS or network drive

This way, even if your main drive fails, your saves are safe elsewhere.

Frequently Asked Questions

Does Checkpoint work with Steam/GOG/Epic games?

Yes, but it's designed primarily for games that DON'T have cloud save sync. Steam, GOG, and Epic already backup your saves to the cloud. Checkpoint is most useful for:

  • Pirated games (not that we endorse that, but it works)
  • GOG games with cloud sync disabled
  • Old games from before cloud saves were common
  • Emulated games (retro console emulators)
  • Games from itch.io or other indie platforms

Will Checkpoint slow down my games?

No. Checkpoint only accesses your save files when you explicitly create a snapshot or restore. It doesn't run in the background monitoring files or using CPU. The only time it's active is when the window is open and you're using it.

Can I use Checkpoint for games with anti-cheat?

Generally yes, but with caution. Checkpoint only copies files in your save directory. It doesn't modify game files, inject code, or do anything that would trigger anti-cheat. However, some aggressive anti-cheat systems might flag any file operations near game files. If you're unsure, check with the game's community first.

How much disk space do I need?

Each snapshot is a complete copy of your save folder. If your saves are 500MB and you create 10 snapshots, that's ~5GB. Plan accordingly. You can always delete old snapshots to free space.

Can I move snapshots between computers?

Yes! The snapshot folders are self-contained. You can copy them from one computer's checkpoint folder to another. Just make sure the game is installed at the same save location path on both machines.

Is my data safe?

Checkpoint never sends your save files anywhere. Everything stays local on your machine (or wherever you set your backup location). The app doesn't connect to the internet except to check for updates (which you can disable).

Troubleshooting

"Failed to create snapshot" Error

Cause: Checkpoint can't read the save files or write to the backup location.

Solutions:

  • Make sure the game isn't running (some games lock save files while running)
  • Check that you have read permissions on the save folder
  • Verify you have write permissions and disk space at the backup location
  • Try running Checkpoint as administrator (Windows) or with sudo (Linux)

"Game is currently running" Warning

You provided an executable name when adding the game, and Checkpoint detected it's running. Close the game completely (check Task Manager/System Monitor) before restoring. Some games have launcher processes that stay open - make sure those are closed too.

Snapshots Taking Forever

Large save files (several GB) take time to copy. This is normal for games with large worlds or many mods. The loading overlay shows it's working. Don't close Checkpoint during this time.

Restored Save Not Showing In-Game

Make sure you:

  1. Restored while the game was completely closed
  2. Waited for the restore to complete (saw success message)
  3. Didn't have the game running in the background

Some games cache save data in memory. If the game was running, it might not see the restored files until restarted.

Tips & Tricks

Naming Conventions

Develop a naming system for snapshots that makes sense to you:

  • Chapter-based: "Ch3 Complete"
  • Location-based: "Before Final Dungeon"
  • Choice-based: "Joined Faction A"
  • Date-based: "2024-02-09 Evening"
  • Build-based: "Melee Build Lvl 50"

The "Before" Rule

Always create a snapshot BEFORE:

  • Major boss fights
  • Story choices that can't be undone
  • Trying out mods
  • Experimenting with game files
  • Starting a new playthrough (to preserve old saves)

Clean Up Regularly

Snapshots take up space. Every few months, review your snapshots and delete:

  • Old auto-backups ("Auto-backup before restore")
  • Snapshots from completed playthroughs you won't return to
  • Duplicate snapshots (same point in game)

Multiple Save Strategies

Use Checkpoint alongside in-game save slots for maximum protection:

  • In-game saves for quick load/save during play
  • Checkpoint snapshots for major milestones and backups

Share Saves With Friends

Want to share a cool save state? The snapshot folder is self-contained. Zip it up and send it to a friend. They can place it in their Checkpoint folder and restore it (make sure they have the same game version).

Pro Tip: Set up a recurring reminder to create a snapshot weekly. It's a 5-second habit that could save you hours of lost progress.
